## Why we use GitHub

We use GitHub to:
- store and organize code, notes, and project files
- track changes over time
- collaborate without overwriting each other’s work
- review updates through pull requests
- keep project history documented

## Basic GitHub workflow for this group

Our typical workflow is:
1. **Pull the latest version** of the repository
2. **Create a branch** for your changes
3. **Make your edits**
4. **Commit** your work with a clear message
5. **Push** your branch to GitHub
6. **Open a pull request**
7. **Request review** if needed
8. **Merge** after approval

## Recommended first-time setup

If this is your first time using GitHub on your computer:
1. Create a GitHub account
2. Install Git
3. Configure your name and email in Git
4. Clone the repository to your machine
5. Practice making a small branch and pull request

Example setup commands:

```bash
git config --global user.name "Your Name"
git config --global user.email "your_email@example.com"
git clone https://github.com/[ORG-OR-USERNAME]/[REPO-NAME].git
cd [REPO-NAME]
