Spring-Grid K Regression Report (10x10 Random, 3 Models)

Setup

  • Grid: 10x10 (90 free DOFs after pinning the bottom row)
  • Springs: horizontal, vertical, and diagonal pairs with constants ~ Uniform[0.5, 1.5]
  • M = I, C = 0, no external forcing
  • Trajectories: 4, dt = 0.01, t_end = 20.0, total samples = 8004
  • Train/test split: 6403 / 1601 samples (80/20 over pooled samples)
  • Library: linear (theta = u), recovered K = -XiT, symmetrised for physical diagnostics
  • Regression locality assumption: 8-connected neighbours (H + V + both diagonals) + self. The data generator only places springs on H, V, and the NW-SE diagonal, so the regressor is told nothing about the anti-diagonal being zero -- it should discover that from the data.
  • DOF reduction: 700 / 8100 entries (8.6%) of Xi are free parameters; the rest are clamped to zero by mask.

Presentation diagnostics

Detailed rollout / local-parameter / modal diagnostics are generated for the selected presentation models only: STLS dense (lam=0.05), SINDy Adam local (no L1), and SINDy Adam local trajectory (no L1)

This codebase's spring-grid model is scalar-valued: each node carries one out-of-plane displacement. The requested "quiver" visualisation is therefore adapted as a vertical arrow rendering of scalar deflection on the lattice.

Optimization note: The residual Adam local model minimizes acceleration residual MSE on pooled samples. The trajectory Adam local model minimizes full state-trajectory MSE on stacked [u, v] trajectories, using a 3-train / 1-test trajectory split and every 10th sample from the simulated time grid.

Output locations and assumptions

  • Static presentation figures saved under results\figures\spring_grid_10x10_random_three_model.
  • Animations saved under results\animations\spring_grid_10x10_random_three_model.
  • Trajectory diagnostics use the first simulated rollout from generate_dataset(...) and compare it against a rollout of the symmetrised estimated stiffness model from the same initial condition.
  • Node selection used for Figure 3 / Figure 5:
    • corner: node 99 @ (row=9, col=9)
    • edge: node 98 @ (row=9, col=8)
    • interior1: node 44 @ (row=4, col=4)
    • interior2: node 51 @ (row=5, col=1)
  • Figure 5 tracks local 8-neighbour couplings around each selected node. The true system only contains H / V / NW-SE diagonal springs, so the anti-diagonal true reference is zero where applicable.
  • Figure 9 uses modal comparison rather than a dispersion relation because the current codebase uses a finite, heterogeneous lattice rather than a periodic homogeneous medium.

Notes

The "locality assumption" (only neighbour pairs and self-couplings have non-zero K entries) is encoded by the sparse-regression objective: STLS and L1/proximal methods drive non-neighbour entries to zero. The dense SINDy Adam (no L1) baseline is included for comparison and is expected to be more accurate per coefficient but less sparse. For the new physical diagnostics, the estimated operator is interpreted through the symmetrised stiffness matrix K_pred = 0.5 * ((-Xi^T) + (-Xi^T)^T).
