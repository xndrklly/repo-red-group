# SINDy and Neural ODE: Mathematical Description

This document describes the mathematics underlying the SINDy (Sparse Identification of Nonlinear Dynamics) and Neural ODE implementations in this codebase.

---

## 1. Problem Statement

Given time-series measurements of a dynamical system's state **x**(t), discover the governing equations:

$$\frac{d\mathbf{x}}{dt} = \mathbf{f}(\mathbf{x})$$

where $\mathbf x = [x_1, x_2, ..., x_n]^T$ is the state vector with n variables, and **f** is an unknown nonlinear function.

The key assumption is that **f** is **sparse** in a suitable basis of candidate functions. That is, only a few terms from a large library of possible functions actually appear in the true dynamics.

---

## 2. Data Collection

We observe the state at m discrete time steps:

$$\mathbf{X} = \begin{bmatrix} \mathbf{x}(t_1)^T \\ \mathbf{x}(t_2)^T \\ \vdots \\ \mathbf{x}(t_m)^T \end{bmatrix} \in \mathbb{R}^{m \times n}$$

and compute (or estimate) the corresponding time derivatives:

$$\dot{\mathbf{X}} = \begin{bmatrix} \dot{\mathbf{x}}(t_1)^T \\ \dot{\mathbf{x}}(t_2)^T \\ \vdots \\ \dot{\mathbf{x}}(t_m)^T \end{bmatrix} \in \mathbb{R}^{m \times n}$$

### 2.1 Derivative Estimation

Three methods are available:

**4th-order central finite differences** (for clean, evenly-spaced data):

$$\dot{x}(t_i) = \frac{-x(t_{i+2}) + 8x(t_{i+1}) - 8x(t_{i-1}) + x(t_{i-2})}{12 \Delta t} + O(\Delta t^4)$$

This loses 4 samples (2 at each boundary). Implemented in `finite_difference_4th`.

**Direct ODE evaluation** (when the true RHS is known, e.g., for synthetic data):

$$\dot{\mathbf{x}}(t_i) = \mathbf{f}(\mathbf{x}(t_i))$$

No finite-difference approximation error and no boundary data loss. If the states were generated by numerical ODE integration, the state accuracy still depends on the solver tolerances. Implemented in `autograd_derivative`.

**Total variation regularized differentiation** (for noisy data). Solves the optimization problem:

$$\min_u \; \frac{1}{2} \| A u - \mathbf{x} \|_2^2 + \alpha \int |u'| \, dt$$

where A is the antidifferentiation operator, u is the derivative to be estimated, and alpha controls the smoothness. The TV term preserves sharp features while suppressing noise. Solved via iterative preconditioned conjugate gradient. Implemented in `tv_reg_diff`.

---

## 3. Library Construction

We construct a library matrix **Theta** whose columns are candidate nonlinear functions of the state, evaluated at each time step:

$$\boldsymbol{\Theta}(\mathbf{X}) = \begin{bmatrix} | & | & | & | & | & & | \\ \mathbf{1} & \mathbf{X} & \mathbf{X}^{P_2} & \mathbf{X}^{P_3} & \cdots & \cdots & \sin(\mathbf{X}) \\ | & | & | & | & | & & | \end{bmatrix} \in \mathbb{R}^{m \times p}$$

where p is the total number of candidate functions and each column is one candidate evaluated at all m time steps.

### 3.1 Polynomial Terms

For n state variables and maximum polynomial order d, the library includes all monomials:

- **Order 0:** 1 (constant)
- **Order 1:** x_1, x_2, ..., x_n
- **Order 2:** x_1^2, x_1 x_2, ..., x_1 x_n, x_2^2, x_2 x_3, ..., x_n^2
- **Order k:** all products x_{i_1} x_{i_2} ... x_{i_k} with i_1 <= i_2 <= ... <= i_k

The constraint i_1 <= i_2 <= ... <= i_k avoids redundant terms (e.g., x_1 x_2 and x_2 x_1 are the same monomial). This is computed using combinations with replacement.

The number of polynomial terms of order k in n variables is:

$$\binom{n + k - 1}{k}$$

Total library size for polynomial order d:

$$p_{\text{poly}} = \sum_{k=0}^{d} \binom{n + k - 1}{k}$$

Examples:
- n=2, d=5: p = 21
- n=3, d=5: p = 56
- n=4, d=5: p = 126

Implemented in `PolynomialLibrary`.

### 3.2 Fourier Terms

For periodic systems, the library can include trigonometric functions:

$$\sin(k \, x_j), \quad \cos(k \, x_j) \qquad k = 1, 2, \ldots, K; \quad j = 1, 2, \ldots, n$$

This adds 2Kn columns to the library. Implemented in `FourierLibrary`.

### 3.3 Composite Libraries

Multiple libraries can be concatenated:

$$\boldsymbol{\Theta}_{\text{composite}} = \begin{bmatrix} \boldsymbol{\Theta}_{\text{poly}} & \boldsymbol{\Theta}_{\text{fourier}} & \boldsymbol{\Theta}_{\text{custom}} \end{bmatrix}$$

Implemented in `CompositeLibrary`. This is the primary extensibility mechanism: any function mapping x -> scalar can serve as a library column.

## 4. Sparse Regression

The system identification problem is now a linear regression with a sparsity constraint:

$$\dot{\mathbf{X}} = \boldsymbol{\Theta}(\mathbf{X}) \, \boldsymbol{\Xi}$$

where the coefficient matrix **Xi** has shape (p x n). Each column of **Xi** gives the equation for one state variable. Sparsity means most entries of **Xi** are exactly zero.

### 4.1 Sequential Thresholded Least Squares (STLS)

The classical SINDy algorithm. Non-differentiable, fast, deterministic.

**Algorithm:**

1. Initialize with least-squares solution:

$$\boldsymbol{\Xi}^{(0)} = \boldsymbol{\Theta}^{\dagger} \dot{\mathbf{X}}$$

where the dagger denotes the pseudoinverse (computed via `lstsq`).

2. For iterations k = 1, 2, ..., K (default K=10):

   a. **Threshold:** identify small coefficients:

   $$S = \{ (i,j) : |\Xi_{ij}^{(k-1)}| < \lambda \}$$

   Set them to zero: Xi_{ij} = 0 for all (i,j) in S.

   b. **Re-solve:** for each state variable j = 1, ..., n, let B_j = {i : (i,j) not in S} be the indices of surviving terms. Re-solve least-squares on the reduced library:

   $$\boldsymbol{\Xi}_{B_j, j}^{(k)} = \boldsymbol{\Theta}_{:, B_j}^{\dagger} \, \dot{\mathbf{X}}_{:, j}$$

The threshold lambda is the key tuning parameter. Larger lambda produces sparser models (fewer terms); smaller lambda retains more terms. Typical values: 0.01-0.3.

Implemented in `stls`.

### 4.2 Gradient-Based Sparse Optimization

PyTorch-specific approach where **Xi** is an `nn.Parameter` optimized via gradient descent.

**Derivative matching objective:**

$$\min_{\boldsymbol{\Xi}} \; \operatorname{MSE}\!\left(\boldsymbol{\Theta} \boldsymbol{\Xi}, \dot{\mathbf{X}}\right) + \lambda \| \boldsymbol{\Xi} \|_1$$

where `MSE` is the mean squared error over all entries, matching PyTorch's `F.mse_loss`, and the L1 penalty promotes sparsity. Solved with Adam by default, or another PyTorch optimizer if supplied.

**Trajectory matching objective (end-to-end differentiable):**

$$\min_{\boldsymbol{\Xi}} \; \operatorname{MSE}\!\left(\hat{\mathbf{X}}(\boldsymbol{\Xi}), \mathbf{X}_{\text{true}}\right) + \lambda \| \boldsymbol{\Xi} \|_1$$

where the predicted trajectory is obtained by integrating the discovered ODE:

$$\hat{\mathbf{X}}(\boldsymbol{\Xi}) = \text{ODESolve}\left( \frac{d\mathbf{x}}{dt} = \boldsymbol{\Theta}(\mathbf{x}) \, \boldsymbol{\Xi}, \; \mathbf{x}_0, \; t \right)$$

For trajectory matching, this package supports three gradient backends:

- `gradient_method="autograd"`: standard PyTorch backpropagation through the ODE solve
- `gradient_method="sensitivity"`: an explicit forward sensitivity solve on the augmented state `(x, S_theta)`
- `gradient_method="adjoint"`: an explicit reverse-time adjoint solve with parameter-gradient accumulation

`ODEModel(use_adjoint=True)` only changes the backend used by `gradient_method="autograd"`; it is not the same as the explicit `gradient_method="adjoint"` option.

**Proximal gradient descent (ISTA):** In a proximal-style training loop, after each gradient step on the MSE term, apply soft thresholding:

$$\Xi_{ij} \leftarrow \text{sign}(\Xi_{ij}) \cdot \max\left( |\Xi_{ij}| - \lambda \eta, \; 0 \right)$$

where eta is the learning rate. This is the proximal operator for the L1 norm.

In the provided `SparseOptimizer` methods, the L1 term is already included in the loss before `backward()`. The separate `proximal_step()` helper is available when using an ISTA-style loop rather than as a required extra step after every built-in optimizer step.

**Hard thresholding:** Periodically during or after training, small coefficients are set exactly to zero:

$$\Xi_{ij} \leftarrow 0 \quad \text{if} \quad |\Xi_{ij}| < \tau$$

Implemented in `SparseOptimizer`.

### 4.3 Neural ODE Alternative

The Neural ODE model uses a neural network instead of a sparse library expansion:

$$\frac{d\mathbf{x}}{dt} = g_{\boldsymbol{\theta}}(\mathbf{x})$$

where $g_{\boldsymbol{\theta}}$ is an MLP with trainable parameters $\boldsymbol{\theta}$. This is more flexible than SINDy, but the learned terms are usually less interpretable because they are encoded in neural network weights rather than explicit library coefficients.

For derivative matching, the objective is:

$$\min_{\boldsymbol{\theta}} \; \operatorname{MSE}\!\left(g_{\boldsymbol{\theta}}(\mathbf{X}), \dot{\mathbf{X}}\right)$$

For trajectory matching, the predicted trajectory is obtained by integrating:

$$\frac{d\hat{\mathbf{x}}}{dt} = g_{\boldsymbol{\theta}}(\hat{\mathbf{x}})$$

and minimizing the trajectory MSE against observed data. Implemented in `NeuralODEModule` and trained with `GradientOptimizer`, which uses Adam by default.

## 5. Model Evaluation

For SINDy, once **Xi** is identified, the discovered dynamical system is:

$$\frac{d\mathbf{x}}{dt} = \boldsymbol{\Theta}(\mathbf{x}) \, \boldsymbol{\Xi}$$

This is evaluated for a single state vector **x** at each ODE solver step. The library Theta(**x**) is constructed from the current state, then multiplied by **Xi** to get the time derivative.

For a 2-variable system with poly_order=2, this expands to:

$$\frac{dx_1}{dt} = \Xi_{1,1} + \Xi_{2,1} x_1 + \Xi_{3,1} x_2 + \Xi_{4,1} x_1^2 + \Xi_{5,1} x_1 x_2 + \Xi_{6,1} x_2^2$$

$$\frac{dx_2}{dt} = \Xi_{1,2} + \Xi_{2,2} x_1 + \Xi_{3,2} x_2 + \Xi_{4,2} x_1^2 + \Xi_{5,2} x_1 x_2 + \Xi_{6,2} x_2^2$$

After sparsification, most Xi entries are zero, yielding concise equations. For Neural ODEs, the corresponding right-hand side is $g_{\boldsymbol{\theta}}(\mathbf{x})$. Both model families are integrated with standard ODE solvers (RK45/dopri5 via `torchdiffeq`).

Implemented in `SINDyModule.forward`, `NeuralODEModule.forward`, and `ODEModel.forward`.

## 6. Trajectory Gradient Methods

For the trajectory matching objective, gradients must flow through the ODE solver. Three approaches are available:

**Direct backpropagation:** Store all intermediate ODE solver states and backpropagate through them. Memory cost: O(L) where L is the number of solver steps. Suitable for small systems.

**Explicit sensitivity method:** Propagate the state and parameter sensitivities forward together:

$$\frac{d\mathbf{S}_{\theta}}{dt} = \frac{\partial \mathbf{f}}{\partial \mathbf{x}} \mathbf{S}_{\theta} + \frac{\partial \mathbf{f}}{\partial \boldsymbol{\theta}}$$

where $\mathbf{S}_{\theta}(t) = \partial \mathbf{x}(t) / \partial \boldsymbol{\theta}$. After the forward solve, the parameter gradient is assembled by contracting $\mathbf{S}_{\theta}(t_n)$ with $\partial \mathcal{L} / \partial \mathbf{x}(t_n)$ at the sampled times.

**Explicit adjoint method:** Augment the system with an adjoint ODE that propagates gradients backward in time. The adjoint state **a**(t) = d(loss)/d(**x**(t)) satisfies:

$$\frac{d\mathbf{a}}{dt} = -\left(\frac{\partial \mathbf{f}}{\partial \mathbf{x}}\right)^T \mathbf{a}$$

and parameter gradients are accumulated as:

$$\frac{d \, \text{loss}}{d \boldsymbol{\theta}} = -\int_{t_1}^{t_0} \mathbf{a}(t)^T \frac{\partial \mathbf{f}}{\partial \boldsymbol{\theta}} \, dt$$

In this codebase, `gradient_method="adjoint"` implements an explicit sampled-checkpoint reverse solve. Separately, `ODEModel(use_adjoint=True)` enables `torchdiffeq`'s adjoint-backprop implementation for the autograd path. The two approaches share the same mathematical objective but are different implementations.

## 7. Worked Example: Lorenz Attractor

**True system:**

$$\dot{x} = \sigma(y - x), \qquad \dot{y} = x(\rho - z) - y, \qquad \dot{z} = xy - \beta z$$

with sigma=10, rho=28, beta=8/3.

**Library** (poly_order=3, n_vars=3): 20 candidate functions.

$$\boldsymbol{\Theta} = [1, \; x, \; y, \; z, \; x^2, \; xy, \; xz, \; y^2, \; yz, \; z^2, \; x^3, \; \ldots]$$

**STLS result** (lambda=0.025): 5 of 20 library terms are active, so 15 candidate terms are unused. The active terms contain 7 nonzero coefficients across the 3 state equations:

| Term | dx/dt | dy/dt | dz/dt |
|------|-------|-------|-------|
| x | -10.000 | +28.000 | 0 |
| y | +10.000 | -1.000 | 0 |
| z | 0 | 0 | -2.667 |
| xy | 0 | 0 | +1.000 |
| xz | 0 | -1.000 | 0 |

This recovers the expected Lorenz equations to the quick-start verification tolerance. The method correctly identifies that only 5 of the 20 candidate library terms are active, with 7 nonzero coefficients across the 60 possible coefficient entries (20 functions x 3 equations).

## 8. Computational Complexity

| Operation | Cost | Notes |
|-----------|------|-------|
| Library construction | O(m * p) | m samples, p candidate functions |
| Least-squares solve | O(m * p^2 + p^3) | Dense solve for m >= p, per state or multi-output |
| STLS (K iterations) | O(K * n * (m * p_a^2 + p_a^3)) | p_a is the active feature count per state |
| Library evaluation (single state) | O(p) | Inside ODE solver |
| ODE integration (L steps) | O(L * p * n) | L solver steps |
| Adjoint backprop | O(L * p * n) | Same as forward, O(1) memory |

| Symbol | Meaning | Shape |
|--------|---------|-------|
